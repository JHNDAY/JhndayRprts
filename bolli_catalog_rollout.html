Recommended Rollout Strategy

Phase 1: Pilot Study (Current Semester)

Objective: Validate tool effectiveness and identify issues before broad deployment

Structure (Based on David Rattner's Suggestion):

  • Participants: 15-20 members from Curriculum Committee (CC) and Study Group Steering Committee (SGSC)
  • Group A (Control): 7-10 users employing traditional Google Doc catalog
  • Group B (Test): 7-10 users employing AI-enhanced visualization tool

Data Collection:

  • Time spent browsing catalog
  • Number of courses explored vs. registered
  • User satisfaction ratings (1-5 scale)
  • Specific pain points or confusion areas
  • Feature requests and suggestions

Success Metrics:

  • Reduced browsing time by 30%+
  • Higher satisfaction scores (4.0+ avg)
  • Zero critical errors or misinformation
  • Positive feedback from 75%+ of participants

Phase 2: Limited Release (Fall 2026)

Objective: Expand access to tech-comfortable members while monitoring performance

Implementation:

  • Offer tool as optional alternative alongside traditional catalog
  • Target audience: Members who indicated interest in technology sessions
  • Provide comprehensive user guide (already created)
  • Establish feedback mechanism (survey link, email contact)
  • Monitor usage analytics and error reports

Support Structure:

  • Drop-in help sessions led by Technology Committee
  • Video tutorial (5-7 minutes)
  • Quick reference guide (1-page PDF)
  • Dedicated email for questions/issues

Phase 3: Full Deployment (Spring 2027)

Objective: Make enhanced catalog the primary navigation tool

Approach:

  • Position as official catalog interface with Google Doc as backup
  • Include brief introduction in member orientation materials
  • Incorporate enhancements from Kate's suggestion (calendar overview)
  • Continue offering traditional format for those who prefer it

Key Enhancement: Calendar Overview

Kate Ehrlich's suggestion for a calendar-based overview addresses a significant user need. This should be prioritized for Phase 2 implementation.

Proposed Calendar Features:

  • Weekly grid view showing all time slots
  • Color-coded by category (History, Arts, Science, etc.)
  • Clickable course titles linking to full details
  • SGL (Study Group Leader) names visible
  • Conflict detection for selected courses
  • Export to personal calendar (iCal/Google Calendar)

Risk Mitigation

HIGH AI Hallucinations/Errors

Mitigation:

  • Manual quality review by 2-3 users before each release
  • Cross-reference tool output against official catalog
  • Include disclaimer: "Verify course details with official catalog"
  • Implement error reporting button on each course page

MEDIUM User Adoption/Comfort

Mitigation:

  • Emphasize tool as "enhanced navigation" not "AI"
  • Maintain traditional catalog as alternative
  • Provide multiple support channels
  • Collect and showcase positive testimonials

LOW Technical Issues

Mitigation:

  • Host on reliable platform (GitHub Pages already used)
  • Test across browsers and devices
  • Maintain offline backup version
  • Document recovery procedures
